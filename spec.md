# BNN 高速推理项目规格（v0.1）

## 1. 目标

基于 C 语言实现一个极快的 BNN（Binary Neural Network）推理模块，对固定长度输入进行高速打分。

- 输入总长度固定：`1088 byte`
- 前 `64 byte`：配置域（config）
- 后 `1024 byte`：业务载荷（payload）
- 输出：单值分数（score），用于快速判定/排序

项目完成后必须完成两类验收：

1. 推理时延（latency）与吞吐（throughput）
2. 准确性（accuracy）与稳定性

核心优先级（从高到低）：

1. 单样本推理时延最小化（目标 `<= 10 us`）
2. 在低时延下保持可接受准确性
3. 代码可维护性与可移植性

## 2. 输入与输出规范

### 2.1 输入布局

- `input[0:64)`：配置域，共 64 字节
- `input[64:1088)`：payload，共 1024 字节

约束：

- 单次推理只接受 `1088` 字节输入，不接受可变长
- 对齐策略默认按 64-bit 对齐优化（实现时可按平台调整）
- 配置域只用于模型路由，不直接参与 BNN 点积
- BNN 计算仅作用于后 `1024B payload`

### 2.2 输出定义

- 输出 `float score`（范围由训练后标定决定）
- 可选输出 `uint8_t label`（例如阈值二分类）

## 3. 模型与实现约束

### 3.1 BNN 基本约束

- 主体计算采用二值权重与二值激活（`{-1, +1}` 或位表示）
- 核心算子使用位运算实现（`XNOR + POPCNT`）
- 优先减少内存访问和分支，保证稳定低时延
- 单模型权重维度固定为 `1024B payload = 8192 bit`（`128 x uint64`）

### 3.2 C 实现要求

- 语言：C11（可兼容 C17）
- 编译优化：`-O3 -march=native`（或对应目标平台参数）
- 必须尽可能使用 Intel AVX 指令集进行加速（AVX2 为基线，AVX-512 为增强路径）
- 代码结构建议：
  - `src/bnn_infer.c`：推理核心
  - `src/bnn_infer.h`：对外接口
  - `src/bnn_weights.h`：静态权重/参数（由训练导出）
  - `src/bench.c`：性能测试
  - `src/eval.c`：准确性评估

### 3.3 Intel 向量化与指令集策略（强制）

- 目标平台优先：`x86_64 Intel CPU`
- 实现要求：
  - 优先使用 `AVX2` 向量路径（256-bit）
  - 若硬件支持，提供 `AVX-512` 优化路径（512-bit）
  - `POPCNT` 必须启用；若可用 `VPOPCNTDQ`，可在 AVX-512 路径使用
- 工程要求：
  - 提供运行时 CPU 特性检测（CPUID），在 `AVX-512 -> AVX2 -> scalar` 间分发
  - 保证不同路径输出一致（误差容忍仅限浮点末位）
  - 基准测试需分别报告各路径性能

### 3.4 多模型路由与热加载（强制）

- 支持运行时加载多个模型权重文件
- 模型选择由前 `64B config` 的位字段决定
- 默认路由规则：读取 `config[0:8]` 后按 `mask/shift` 提取 `model_id`
- 路由未命中时必须回退到默认模型

## 4. 训练与导出流程（与 C 推理解耦）

### 4.1 训练侧

- 训练脚本可使用 Python 框架（如 PyTorch），但最终部署仅使用 C 推理
- 输入样本必须严格映射到 `64B config + 1024B payload`
- 保存训练配置、随机种子、数据切分信息，保证可复现

### 4.2 导出侧

- 将 BNN 权重/偏置/阈值导出为 C 头文件（`*.h`）
- 明确位打包顺序（bit order）和端序（endianness）
- 导出后需做一次 Python 与 C 的逐层一致性校验

## 5. 推理接口定义（初稿）

```c
// 返回 0 表示成功，非 0 表示错误
int bnn_score_1088(const uint8_t input[1088], float *score);
```

可选扩展接口：

```c
int bnn_score_batch_1088(const uint8_t *inputs, size_t n, float *scores);
```

## 6. 性能验收标准（首版）

> 性能优先，`10us` 级别是本项目主目标。

- 单样本 P50 时延：`<= 10 us`（硬目标）
- 单样本 P99 时延：`<= 15 us`（目标）
- 单样本 P99 时延：`<= 20 us`（最低可接受）
- 吞吐：`>= 100 K samples/s`（单线程基线）

测试方法：

- 固定 CPU 频率策略（尽量避免频率漂移）
- 预热后测量（例如预热 1e5 次）
- 正式统计不少于 1e6 次推理
- 记录 P50/P90/P99 与均值/标准差
- 单独记录 `AVX2` 与 `AVX-512` 路径结果（若硬件支持）

### 6.1 冲击 10us 的实现策略（必须执行）

- 算子层：
  - 全路径采用 `XNOR + POPCNT`，避免浮点主计算
  - 热路径循环展开，减少分支与边界检查
- 内存层：
  - 权重连续存储并按 64-byte 对齐
  - 减少 cache miss，优先 L1/L2 命中
  - 避免动态分配（推理热路径中 `malloc/free` 禁止）
- CPU 层：
  - 固定线程绑核（CPU affinity）
  - 关闭频繁上下文切换影响（基准时独占核心）
  - 区分 `AVX2` 与 `AVX-512` 的最优 block 大小
- 工程层：
  - 单样本接口走最短路径，不做冗余校验
  - 批量接口与单样本接口分离优化，避免互相牵制
  - 每次优化必须附带基准数据对比

## 7. 准确性验收标准（首版）

- 主指标：`AUC` 或 `F1`（按业务任务定义）
- 阶段目标：与训练侧参考实现相比，离线评估偏差不超过 `±0.5%`
- 一致性：同一输入多次推理结果完全一致（确定性）
